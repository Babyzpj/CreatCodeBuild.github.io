Questions You Need to Ask Yourself before Starting Deep Learning

I was recentedly asked this question quite frequently, “How to get started on deep learning?”

For some reason I was considered a “deep learning pro” by some of my friends and some abstract friends online. But I merely just got started on deep learning myself! How am I supposed to give advices? But, maybe there is something I can share. I started to recall what have I done to begin my journey of deep learning and there are many things to share.

Let me tell you a little bit about my background. I am a Computer Science student from a technology university. I am studying for my Bechalor degree. Therefore, one thing I am sure is that you don’t need a PhD to do deep learning. However, that also said I had a lot of programming experience and computer science knowledge before I dive into deep learning.

Before you dive into Deep Learning or Machine Learning in general, you need to ask yourself at least 2 questions.

What is your background?

Not your culture background or where you live, but what is your academic or professional background? Are you a college student like I am? If you are, are you a CS student? a math Major? or other less related majors? Maybe you are a software engineer. If so, which field you work on? Web? Mobile? Networking? System? Scientific Computing? Or, maybe you work in other industries which are more far away from the world of computing and softwares.

Your background tells you how much basic knowledge you need to have in order to get started on the real meat, Deep Learning.

If you are a computer science student, a math student, or a engineering/science student or a software engineer, you probably have more Deep Learning required knowledge than people from other background.

But, I am not discouraging people from other background, because the second question is also very important.

What do you want to get out of Deep Learning?

What is your goal? What do you want to do with Deep Learning or Machine Learning in general?

If you background tells you what you have to learn, you goal might tell you what you don’t have to learn. Jason Brownlee has explained this in detail in his great article Find Your Machine Learning Tribe.

To summary, there are 2 kinds of people:

  1. You want to understand the theories and practices of deep learning and push this field forward.
  2. You want to solve some problems at hand and don’t have to understand all the details.

Most people belong to the second group, as do I. I am just a student. I am impatient. I don’t want to wait until I finish reading all textbooks. I want to start learning now.

Skills you must have before any deep learning projects

1. Basic programming
It does not matter which language. But, you need to know what is a variable and what is a function that sort of things. As long as you don’t implement algorithms or frameworks yourself, you don’t have to be a pro coder.

2. Some Math
The math part of deep learning is composed of Linear Algebra, Calculus and Probability/Statistics.

You need to know what is a vector dot product and what is a matrix product. You understand multi variable derivatives and the chain rule. You know what is probability and what a distribution means in statistics, such as gaussian and uniform distribution.

You don’t have to know all the theorems and proofs. You don’t need to do mental calculus.

3. Basic Machine Learning
Deep Learning is an approach, a method of Machine Learning.

You need to know at least the difference between supervised learning and unsupervised learning at a conceptual level. Supervised learning is probably the best way to get started on Deep Learning. For supervised learning, you need to know what is a training set and what is a testing set. What is a loss function. What is classification and what is regression.

You don’t need to know all kinds of algorithms and the math behind it.

That’s bascially all you have to know.

Now, how to get started?

The following 4 steps are not ordered. You almost do them simultaneously.

0. Get a good course

Maybe you are an experienced programmer. But Deep Learning isn’t like GUI application programming, where you can just download a framework, read the docs and you can start programming immediately, which is how most GUI developers start.

No offense to GUI application developers, but Deep Learning requires a fair amount of mathamatical knowledge.

Most frameworks(all frameworks which I have worked with) have great docs. But all of them assume some (theoretical) knowledge of deep learning. If you have these knowledge, you will have docs very helpful. If you don’t, docs are just nonsense.

In this sense, Deep Learning frameworks’ docs are similar to socket programming docs. Socket programming docs in most languages are great docs to show you how to do it (if you know what to do). But few of them are good for learning it.

I recommend 3 recourses which I have all learnt:

  1. Udacity They have free and paid courses. Just search “deep learning” or “machine learning” on their website, there are several.
  2. CS231n from Stanford and Videos on youtube
  3. CS224d from Stanford and Videos on youtube

I believe there many more online for free.

1. Identify your problem

Learn by doing. What kind of problems you are interested in? You can find datasets and problem sets on these website:

  1. Kaggle
    Kaggle is a data scientist competition platform. It has many open dataset to try out. It also host competitions.
  2. OpenAI Gym and OpenAI Universe
    These 2 platforms from OpenAI focus on Reinforcement Learning. It has many problems sets to compete. The most interesting part is that all of its problems are actually video games. Your goal is to build smart models to learn how to play video games.

I am not aware of any other open platforms in English world. Besides these platforms, you can also find other datasets or even make your own data for the unique problem you care about.

2. Find a good framework

There many frameworks, including TensorFlow, Theano, Torch, Caffe, DL4J, CNTK, MXNet….

I personally have only used TensorFlow and Keras at the time when this article is written.

Note that some frameworks are low level, which means its API is at a computational graph or matrix manipulation level. All the deep learning part has to be composed by yourself. You maybe or may not like that.

Some frameworks are actually more like an API specifications, such as Keras which is a high level API using either TensorFlow or Theano as its computational backend.

For beginners, I recommend a high level API. High level API is closer to Deep Learning concepts and it is very important to have the right concepts when a person first starts to learn anything.

High level API also saves you a lot of time from reinventing the wheel. At the beginning, you probably doesn’t need to use cutting edge features anyway.

Also, you may want to choose a framework of which community is active and supportive, so that you can get external helps quickly. Basically, how many people use this framework and how many tutorials online.

3. Get a GPU computer

Deep Learning is all about matrix computation. GPU is born to do it.

If you are a PC game, you probably have a good Nvidia GPU. If not, considering buy one or set up a cloud GPU instance. AWS or Google Clound.

This will save you hours or even months of time. The speed up of GPU could be 10 times or even more than 100 times depending on your application and hardware.

The End

That’s it. In this article, I shared how to get started on Deep Learning at a high level. I didn’t get into details or code because that’s out of the scope of this article. I hope you find it useful.
